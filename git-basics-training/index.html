Git installieren

https://git-scm.com/download

  • MacOS
    • Alternativ per homebrew ( http://brew.sh/ )
    • brew install git
  • Windows
    • Git-Bash für Installation auswählen
    • Nutzung wird empfohlen, da damit die Git-Nutzung unter Windows angenehmer ist
  • GNU/Linux
    • apt-get install git # Debian / Ubuntu
    • yum install git # Fedora
    • pacman -S git # Arch
    • usw.

Arbeitsordner für das Modul WBA1 erstellen

  • Im Terminal / Konsole ( bzw. Git-Bash )
    • per cd <ordnername> und cd .. durch Ordner navigieren
    • Ordner erstellen mit mkdir wba1
    • per cd wba1 in den erzeugten Ordner wechseln

Profil-Datei erstellen

  • Im Ordner employee_profiles eine neue Datei erstellen
  • mit dem folgenden Muster nachmane.vorname.json (eigener Name)
  • Z. B. per Terminal / Konsole: touch nachname.vorname.json
  • Folgendes JSON-Dokument in die soeben erzeugte Datei schreiben 
                    
{
  "surname"     : "Nachname",
  "name"        : "Vorname",
  "work_package": "Arbeitspaket",
  "motto"       : "Motto"
}
  • Direkte Eingabe des JSON-Dokuments über das Terminal / die Konsole kannn mittels vim oder nano erfolgen
    • vim nachname.vorname.json

Änderungen commiten

  • Den aktuellen Stand des lokalen Repos anzeigen lassen (Tipp: status)
  • Änderungen der Staging-Area / dem Index hinzufügen (Tipp: add)
  • Änderungen commiten und eine sprechende Commit-Message wählen (Tipp: commit)
    • Denken Sie an The seven rules of a great git commit message
  • Veröffentlichung der lokalen Änderungen (Tipp: push)
  • ggf. (vorher erst) den aktuellen Stand des Company-Repos holen (Tipp: pull)
